Troubleshooting  Container/Pod Access issues

502 - Bad Gateway Error

You get a Bad Gateway Error when the routing to your service fails - as service is unable to serve the Request.

Possible Reasons
1. Service Failed to Restart
2. Service still in Starting mode when the request arrived (Implement Readiness mode  - Checkout chapter 5)

The following command will display the logs (since 2 hours) from your service logs

$ kubectl logs --since=2h pod-name -n namespace

The following command will display the logs (last 10 lines) from your service logs

$ kubectl logs --tail=10 pod_name -n namespace

Container Error

When the Container is failed to start. Check the Container Events.

Possible Reasons
1. Unable to download the container image
2. Unable to start due to Error in the Start up process (For Ex. Null Pointer giving Server Error)
3. Unable to connect to a DB or other external resource

Checkout the evevnts at the bottom of the Pod Description

$ kubectl describe pods pod-name

Ingress Error

Access issues can happen due bad configuration in the ingress definition. Check out the Ingress logs to see whats the issue. 

$ kubectl logs --since=2h -n namespace ingress-pod-name
